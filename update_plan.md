## 本次踩坑与陷阱警告

- ruff 会提示未使用导入对于仅用于副作用的导入（如 `readline`），需要手动添加 `# noqa: F401` 禁止警告。
- 当在全局作用域已经导入 `yaml`，内部函数中重复导入会导致 ruff 错误"referenced before assignment"，即使代码运行正确，最好还是移除重复导入保持 ruff 干净。
- 实验分支合并后记得删除分支，保持分支列表干净。
- 添加新功能/新命令到 argparse 时，注意不要重复定义 main() 函数，会导致语法错误。编辑前确认文件结构避免重复定义。
- 增量扩展功能可能会产生函数副本（如 `renderMemoryList` 被原版本和标签筛选扩展版本各保存一份），修改功能时必须同时更新两个副本保持行为一致。
- 抽取公共函数后忘记添加导入会导致 NameError，修改后必须立即测试验证，不要等到提交前才发现导入缺失。
- Python 标准库 base64 解码不需要额外浏览器兼容函数，直接使用 `base64.b64decode` 就可以，不用找错实现。
