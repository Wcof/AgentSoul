## 本次踩坑警告

- TypeScript 中 `fs.accessSync` 返回 void，不能用于 if 条件判断，必须用 try-catch
- 表达式索引类型需要显式类型断言，TypeScript 不允许 string 直接索引自定义类型
- 接口属性命名必须保持一致性（snake_case），否则 schema 验证和接口实现不匹配会导致编译错误
- 修改 TypeScript 代码后必须重新 `npm run build`，否则修改不会生效
- 正则表达式字面量必须正确加括号，`.match/` 会语法错误，必须是 `.match(/.../)`
- macOS `date` 命令参数格式与 GNU `date` 不同，脚本需要适配
- 对 Optional 类型要显式断言或者用保护语句，不能直接访问属性
- 泛型类型必须提供类型参数，`dict` 需要 `dict[K,V]`，`list` 需要 `list[T]`，否则 mypy --strict 会报错
- Python 3.9+ 支持内置泛型语法 `list[T]`/`dict[K,V]`，不再需要从 `typing` 导入 `List`/`Dict`
- 创建测试文件时注意目录位置，不要误创建到 `mcp_server/tests` 子目录
- ruff 新版本已废弃顶层 `select`/`ignore` 配置，必须移动到 `[tool.ruff.lint]` 区段
- 编写单元测试时，访问计数计算需要仔细核对初始值，简单加法错误会导致测试失败
- invalidate_cache 后立即检查 `_priorities` 长度，它已经被清空，重载只在下次访问时发生
- ISO 周日历计算时，要注意不同日期对应的实际周日数可能和估算不一致，测试前需要实际验证
- JSON 解码失败需要添加异常处理，不能假设返回内容一定是预期格式，否则会抛出未捕获异常
- 使用 mock 测试时，side_effect 需要小心控制调用顺序，获取正确的参数需要检查 call_args_list
- 文件路径不正确会导致 pytest 找不到测试文件，需要使用正确的相对路径
- zip 导入时，压缩包内容放在子目录内，需要递归查找文件，不能只在根目录查找
- 编写单元测试时，直接导入的全局变量在模块内部重新赋值后，测试文件中保留旧引用，需要通过模块访问
- 偏好学习需要多次积累置信度超过阈值才会更新偏好，单次学习不会更新最终偏好值，测试期望值需要考虑这一点
- 正则分词会把下划线当作单词字符，所以 "tag_1" 会被识别为单个单词，不会拆分出 "tag"
- 计算编辑距离时，"teh" vs "the" 实际编辑距离是 2（两个字符错误），不是 1，选择测试用例需要仔细验证
- **测试缺少 master 配置时，同时也缺少 behavior 配置，会产生两个警告而不是一个，测试期望需要考虑这一点**
- 权限测试需要提前创建所有子目录再修改权限，否则父目录权限修改会导致子目录存在性检查失败
- 归档一致性测试中，日期计算依赖解析后的日期，必须确保文件名匹配正确解析模式才会触发警告
- TTL 缓存最小为 1 秒，小于 1 会被自动钳位，测试需要覆盖边界情况
