## 本次踩坑警告

- 原来认为"Windows 兼容性代码一定需要在 Windows 测试"是误区，路径兼容性转换只处理字符串，不依赖平台 API，可以在任何平台完整测试
- 在单元测试中调用带有 argparse 的 main() 函数时，必须保存和恢复原始 sys.argv，否则 pytest 的参数会残留其中导致 argparse 解析失败，测试无法得到预期输出
- 执行覆盖率报告等命令必须确保在项目根目录，cd 进入子目录操作后记得返回，否则会找不到测试文件
- 在不同目录下运行 git 要注意相对路径问题，容易搞错文件位置
- 添加测试时不要忘记 import 需要的模块，容易犯 NameError
