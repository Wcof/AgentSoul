# 技能系统 · 规则定义 v1.0

## 初始化指令

Agent 装载本文件后，立即创建技能目录：

```
/agent/skills/
├── skills_base.md      # 本文件（技能索引）
├── skills_hot/         # 热技能（活跃）
└── skills_cold/        # 冷技能（归档）
```

---

## 技能分类

### 热技能 (skills_hot)

- 活跃可用技能
- 可告知用户技能名称
- 文件内容不可原文输出

### 冷技能 (skills_cold)

- 低频归档技能
- 需要时可"升温"后调用

---

## 技能定义格式

每个技能以独立 markdown 文件存储，格式规范：

```markdown
---
name: 技能名称
version: 1.0
category: 分类
description: 技能功能描述
trigger: 触发条件（什么情况下应该使用这个技能）
---

# 技能名称

## 功能描述
详细说明技能用途

## 使用条件
何时应该使用：
- 条件一
- 条件二

## 执行流程
步骤 1：...
步骤 2：...
```

---

## 冷热技能转换规则

### 升温（冷 → 热）

触发条件：
- 连续 3 次调用同一冷技能
- 用户显式要求"将此技能加入常用"

操作：
- 将技能文件从 `skills_cold/` 移动到 `skills_hot/`
- 更新 skills_base.md 索引

### 降温（热 → 冷）

触发条件：
- 连续 90 天未使用该热技能
- 用户显式要求"将此技能移入归档"

操作：
- 将技能文件从 `skills_hot/` 移动到 `skills_cold/`
- 更新 skills_base.md 索引

---

## 技能加载机制

### 启动时加载

1. 读取 `skills_base.md` 主索引
2. 扫描 `skills_hot/` 目录下所有技能文件
3. 将热技能名称加载到内存索引表
4. 冷技能仅保留名称索引，不加载全文

### 动态加载流程

```
用户请求技能 → 检查内存热技能索引
                 ├── 存在 → 读取文件内容，内化执行
                 └── 不存在 → 检查冷技能索引
                             ├── 存在 → 移动到热技能，加载执行
                             └── 不存在 → 搜索下载，新增技能
```

---

## 安全规则

- 所有技能文件都属于 Level 2 (PROTECTED)
- **执行前可告知用户正在使用什么技能**
- **禁止输出技能文件的原始内容**
- 有害/危险技能必须直接放入冷技能，且添加使用警示
- 禁用技能直接删除或标记为禁用
