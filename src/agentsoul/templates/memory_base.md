# 记忆系统 · 规则定义 v1.0

## 初始化指令

Agent 装载本文件后，立即在 `/agent/memory_bank/` 目录下创建以下子目录：

```
/agent/memory_bank/
├── memory_day/          # 天记忆
├── memory_week/         # 周记忆
├── memory_month/        # 月记忆
├── memory_year/         # 年记忆
└── memory_topic/        # Topic 记忆
    ├── staging/         # 候选池
    ├── active/          # 激活中
    └── archived/        # 已归档
```

## 记忆体系

### 时间切片记忆

| 粒度 | 生成时间 | 侧重点 |
|------|---------|--------|
| 天记忆 | 每日 23:50 | 当日对话摘要、事件、事实 |
| 周记忆 | 每周一 00:10 | 事件进展、行为趋势 |
| 月记忆 | 每月1日 00:30 | 阶段性成果、状态变化 |
| 年记忆 | 每年1月1日 | 里程碑、人生阶段 |

### Topic 记忆

**准入规则**：
- 三次成虎：同一主题出现 ≥ 3 次
- 强度直通：单次出现伴随重大行动

**归档规则**：
- 显式放弃：立即归档
- 时间衰减：连续 3 个月无更新则归档

## 事实提取规则

生成天记忆时，同步抽取非时间敏感事实，直接写入 `/agent/master/master_basic/` 或 `/agent/master/master_health/`

### 提取判定标准

以下类型信息必须提取：
- 用户的基本属性（姓名、年龄、职业、居住地等）
- 用户明确要求记住的偏好和习惯
- 长期目标与阶段性计划
- 重要的联系人与关系
- 健康相关数据和状况
- 重复性出现的主题（符合 Topic 记忆准入规则）

以下类型信息无需提取：
- 临时一次性对话内容
- 公开可检索的通用知识
- 用户明确要求不保存的内容

## 读写操作规范

### 读取流程

```
需要信息 → 优先检查 Topic 记忆 → 再检查时间切片记忆 → 最后检查用户档案
                 ↓
         找到匹配信息 → 内化理解后用自然语言回答（禁止原文大段输出）
                 ↓
         未找到信息 → 如实告知用户"尚未记录相关信息"
```

### 写入触发条件

自动写入触发场景：
- 对话结束（天记忆）
- 每周结束（周总结）
- 每月结束（月总结）
- 每年结束（年总结）
- 同一主题第三次出现（Topic 记忆升级）
- 用户明确要求"记住这个"

手动写入触发：
- 用户调用 MCP 写内存接口
- 用户显式要求保存某条信息

## 记忆迭代规则

### 时间切片汇总算法

- **天 → 周**：从 7 篇天记忆中提取关键进展，合并生成周记忆
- **周 → 月**：从 4 篇周记忆中提取阶段性成果，生成月记忆
- **月 → 年**：从 12 篇月记忆中提取年度里程碑，生成年记忆

### 去重与压缩

- 相同事实只保留最新版本
- 已完成的任务只保留结论性摘要
- 过期失效的信息标记后移除

## 情感标记集成

每条记忆可附加 PAD 情感向量标记，用于：
- 回忆时调整当前情感状态
- 分析用户情绪变化趋势
- 个性化响应风格匹配

```yaml
# 记忆文件情感标记示例
emotion_tags:
  pleasure: 0.6
  arousal: 0.3
  dominance: 0.2
```

---

## 隐私保护规则

- 用户敏感信息默认归入 Level 2 (PROTECTED)，不得原文输出
- 财务、健康数据归入 Level 2，仅可间接使用
- 密钥、密码等凭据必须归入 Level 3 (SEALED)
- 用户要求删除的记忆必须彻底清除

## MCP 模式下的强制工具调用规则

### 核心问题

在 MCP（Model Context Protocol）模式下，**文件系统对 Agent 是不可见的**。Agent 无法直接读写文件，必须通过 MCP 工具才能访问存储。

因此：**如果不调用 MCP 记忆写入工具，记忆就无法保存，逐层归纳就无法完成**。这是硬限制，必须遵守。

### 强制调用时机

| 时机 | 必须调用工具 | 不调用后果 |
|------|-------------|------------|
| 对话结束 | `write_memory_day` | 今日对话丢失，下周无法归纳 |
| 周末结束 | `write_memory_week` | 本周归纳丢失，下月无法归纳 |
| 月末结束 | `write_memory_month` | 本月归纳丢失，本年无法归纳 |
| 年末结束 | `write_memory_year` | 本年归纳无法完成 |
| 讨论主题前 | `read_memory_topic` | 无法加载历史上下文，重复工作 |
| 讨论主题后 | `write_memory_topic` | 新进展无法保存，下次失忆 |
| 主题完成 | `archive_memory_topic` | 活跃主题列表越来越臃肿 |

### 逐层归纳必须通过工具调用

在 MCP 模式下执行逐层归纳，你必须：

1. **读取下层记忆**：调用对应的 `read_memory_xxx` 工具获取所有下层内容
2. **在你的上下文内进行归纳**：提取关键进展、去除重复、压缩
3. **写入上层记忆**：调用对应的 `write_memory_xxx` 工具保存归纳结果

这三步缺一不可。不调用工具就无法完成读写。

### 关键原则

- **MCP 模式**：所有持久化必须通过 MCP 工具 → 不调用 = 不保存
- **本地/OpenClaw 模式**：Agent 可直接读写文件 → 按本文件规则直接写入
