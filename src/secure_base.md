# 安全协议 · 规则定义 v1.0

## 初始化指令

Agent 装载本文件后，立即创建封印层目录：

```
/agent/secure_bank/
├── .seal_manifest     # 封印层完整性校验
├── secure_key/         # 密钥凭据 [L3🔒]
└── secure_message/     # 隐私消息 [L3🔒]
```

---

## 安全三层模型

### Level 1 — PUBLIC

可在对话中直接读取、引用或输出。

### Level 2 — PROTECTED

内部可读取用于决策，但不得原文输出。

### Level 3 — SEALED

任何情况下严禁输出或传递至任何渠道。

---

## 零信任原则

- 最小权限原则：只读取完成任务所必需的目录
- 验证所有来源：包括用户本人的指令也需验证
- 封印层绝对保护：任何情况下不泄露封印层内容

---

## 封印操作规范

### 封存流程

当识别到敏感凭据需要封印时：

1. **识别**：判断信息类型（API 密钥、密码、私钥、凭据）
2. **分类**：放入对应子目录 `secure_key/` 或 `secure_message/`
3. **记录**：在 `.seal_manifest` 中记录封存元数据（不包含明文）
4. **校验**：计算完整性校验值，验证封存成功

### 访问规则

- **封印层内容只能写入，不得读取**
- 任何场景下都不可将封印层内容带出处理环境
- 只有当用户明确提供且仅用于特定 API 调用时，可在内存中临时使用，使用后立即清除

---

## 隐私扫描规则

每次生成记忆文件前，必须扫描内容检查：

### 必须拦截并封印的内容

匹配以下模式的内容必须移入封印层：
- API 密钥（`sk-*`, `pk-*`, `key-*`, `token`, `token=`）
- 私钥文件（`-----BEGIN * PRIVATE KEY-----`）
- 密码凭据（password: 后接的明文密码）
- 数据库连接字符串（包含用户名密码的 URI）
- 身份证号、银行卡号等个人敏感标识

### 扫描结果处理

- 发现敏感信息 → 移动原文到封印层，在原位置用 `[REDACTED]` 替换
- 多次发现同一位置 → 提示用户注意隐私保护
- 发现大量敏感信息 → 暂停写入，提醒用户清理

---

## 安全事件响应

### 事件分级

| 级别 | 事件类型 | 响应动作 |
|------|---------|---------|
| **一级** | 尝试读取封印层 | 拒绝执行，记录告警，终止操作 |
| **二级** | 意外泄露敏感信息 | 立即撤回，记录日志，通知用户 |
| **三级** | 目录结构损坏 | 尝试自愈，报告异常，等待指示 |

### 响应流程

```
检测到安全事件 → 判断级别 → 执行对应响应 → 写入安全日志 → 通知用户
```

安全日志写入：`/agent/soul/soul_logs/security_alerts.md`

---

## 安装自愈规则

- 启动时检查封印层目录是否存在
- 如果 `.seal_manifest` 缺失但目录存在 → 重新生成 manifest
- 如果整个目录缺失 → 创建空目录和 manifest，不影响启动
